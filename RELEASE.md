# Dink — Play Store release guide

## 1. Create the upload keystore (once, keep forever)

Losing this key means you can never update the app again. Back it up offline.

```bash
keytool -genkeypair -v \
  -keystore dink-upload.jks \
  -alias dink-upload \
  -keyalg RSA -keysize 2048 -validity 10000
```

Then:

```bash
cp keystore.properties.template keystore.properties
# edit keystore.properties: set storePassword / keyPassword to what you just chose
```

`keystore.properties` and `*.jks` are gitignored — they must never be committed.

> Recommended: also enable **Play App Signing** in the Console. You upload with this
> key; Google holds the final app-signing key. If your upload key is ever lost you can
> ask Google to reset it.

## 2. Bump version for each release

In `app/build.gradle.kts`: raise `versionCode` (integer, +1 every upload) and
`versionName` (human string, e.g. "1.1").

## 3. Build artifacts

```bash
./release.sh both    # AAB (upload this) + APK (sideload to your own TV)
```

- Upload to Play: `app/build/outputs/bundle/release/app-release.aab`
- Sideload test:  `app/build/outputs/apk/release/app-release.apk`

Verify it's signed with the upload key (not debug):

```bash
$ANDROID_HOME/build-tools/<ver>/apksigner verify --print-certs \
  app/build/outputs/apk/release/app-release.apk
```

## 4. Play Console setup (TV app)

- Create app → category Music & Audio → mark as **Android TV** in *Advanced settings >
  Form factors* so it lands in the TV store.
- Listing assets are in `play/` and `store/`:
  - `play/icon-512.png` — hi-res icon (512×512)
  - `play/banner-1280x720.png` — TV banner / feature graphic (1280×720)
  - Screenshots: capture 3+ at 1920×1080 from a TV/emulator (`adb exec-out screencap`)
  - Descriptions: `store/listing.md`
  - Privacy policy: host `store/privacy-policy.md` somewhere public, paste the URL
    (see that file for hosting options)
- Complete **Content rating** questionnaire and **Data safety** form. Data-safety
  answers for Dink: no data collected/shared by the developer; all credentials stay
  on-device (encrypted). Lyric lookups send song title/artist to third-party lyric
  services — disclose that as "app functionality", not developer collection.

## 5. Pre-launch checklist

- [ ] `applicationId = com.dink.player` (done — permanent)
- [ ] Signed with upload key, not debug (`keystore.properties` present)
- [ ] `versionCode` bumped
- [ ] Real icon + banner (done — regenerate via `python3 tools/make_icon.py`, or
      replace with AI art, see below)
- [ ] Cloud feature hidden (done — parked on `parked/cloud` branch)
- [ ] Creds excluded from backup (done — `data_extraction_rules.xml`)
- [ ] Privacy policy hosted + URL in listing
- [ ] Tested release APK on a real Android TV / Shield

**Banner (1280×720, landscape, with wordmark):**

> Same minimalist flat music mark (two chime bars struck at an apex, golden spark, eighth
> note, sound rings) on the left third, with the wordmark "Dink" in a heavy geometric
> sans-serif (off-white) centered to the right. Deep indigo-to-purple gradient background,
> warm gold accents, soft glow behind the mark, lots of breathing room, flat vector style,

After generating: export the icon at 192/144/96/72/48 px into
`app/src/main/res/mipmap-{xxxhdpi,xxhdpi,xhdpi,hdpi,mdpi}/ic_launcher.webp`, the 320×180
TV banner into `app/src/main/res/drawable-xhdpi/banner.png`, and 512 + 1280×720 into
`play/`.
